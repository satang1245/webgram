## 2. 장점

### 2.1 정적 타입의 지원.
```javascript
function sum(x,y) {
    return x+y
}

// 의도치 않는 인수를 전달.
sum('1',3); // result : 13
// or
sum('a','b') // result : ab
```
위 함수를 정의한 개발자의 의도는 숫자 타입 인수를 전달받아 합계를 반환하려는 것으로 추측되나,  
코드상에 어떤 타입의 인수를 전달해야 하는지, 어떤 타입의 값을 리턴해야 하는지 명확히자 않아  
의도치 않는 결과를 호출할 수 있다. 하지만 문법상 오류가 있지 않기 때문에, 별다른 오류 없이 출력이 되고 있다.
이러한 변수 및 반환값의 타입을 사전에 명시하지 않고 작성되는 코드를 동적타이핑(Dynamic typing)이라 한다.

```typescript
function sum(x: number, y: number) {
    return a + b;
}

// 의도치 않는 인수를 전달할 경우. 오류가 출력.
sum('1',3); // Argument of type 'string' is not assignable to parameter of type 'number'.
```

타입스크립트를 이용하여 타입을 미리 지정하여 다시 작성하게 된다면,  
위와 같이 오류를 컴파일 단계에서 미리 알 수 있다.   
또한 타입을 미리 지정함으로써, 제작자의 의도를 명확히 알 수 있는 장점이 있다.  
이러한 타입들을 미리 지정하는 것을 정적 타입이라 한다.

## 3. 개발환경
타입스크립트는 .ts 파일로 제작이 가능하지만, 해당 파일은 브라우저에서 동작하지 않기 때문에,  
이를 브라우저가 해석가능하도록 js로 다시 변환하는데, 이를 컴파일 또는 트랜스파일링이라 한다.  
<small>*컴파일을 이용할 node및 npm관련 사항은 여기에서는 다루지 않을꺼임.</small>

### 3.1 컴파일러
node npm을 통하여 typescript의 설치가 가능하며,  
컴파일은 `tsc` 명령어를 통하여 컴파일이 가능하다.
```bash
# npm명령어를 통하여 typescript를 설치
$ npm install typescript

# tsc 명령어를 통하여 .ts파일을 .js로 컴파일.
$ node ./node_modules/.bin/tsc ./hello

# hello.ts -> hello.js 파일 생성.
```
tsc 설정 옵션 참고: [tsc CLI Options](https://www.typescriptlang.org/docs/handbook/compiler-options.html)
